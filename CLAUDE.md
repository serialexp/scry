# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`scry` is an opinionated single-binary replacement for the Grafana observability stack (Loki + Tempo + Mimir + Pyroscope). It stores metrics, logs, traces, and profiles as immutable parquet blocks in S3-compatible object storage and serves queries from the same bucket. One deployment, one tenant, no hash ring, no service split.

Status: **v0.9 complete — scry is now multi-instance: 1–N identical instances share one bucket, coordinated through Valkey (a lease for single-winner compaction/retention + pub/sub for catalog convergence), with a correct single-instance fallback when Valkey is absent. v0.8 (size-tiered compaction + per-signal retention) and all earlier milestones are sealed. v1.0 (Grafana adapters / own UI) is the next round.** A record goes in via the wire protocol (or through `scry gateway`, now a **fan-out hub** that accepts foreign push protocols — OTLP traces, Pyroscope profiles, Prometheus remote-write — *and* the native wire (opt-in `--listen-wire`), then tees every accepted record best-effort to all configured sinks — any of scry / Grafana Loki / OpenSearch, each opt-in (Loki/OpenSearch logs-only); a logs-only gateway needs no scry server at all — D-041), lands in a per-writer WAL, gets uploaded as a parquet block (with a per-signal postings sidecar for metrics/logs and a per-block `body.bloom` full-text skip sidecar for logs; traces/profiles carry neither) to Garage, shows up in a SQLite catalog (online or reconciled from the bucket), and is queryable through DataFusion either locally (`scry get`) or over the `scry query` daemon's binschema-framed wire. v0.1 (storage), v0.2 (metrics ingest+query), v0.3 (query daemon), v0.4 (logs), the unnumbered push-gateway milestone, v0.5 (traces query: `--trace-id` by-id lookup + promoted-column matchers, predicate pushdown), v0.6 (profiles retrieval query: raw pprof blob out), and v0.7 (full-text log search: `--grep`/`body_contains` accelerated by an inline byte-trigram bloom skip sidecar) are all sealed by `scripts/smoke.sh`, which asserts the full ingest → store → **query** round-trip per signal (plus a `--trace-id` lookup for traces and a `--grep` ≡ `body LIKE` equivalence + bloom-sidecar check for logs). v0.8 adds **compaction** and **retention**. Compaction: the standalone `scry compact` tool (`--once` / `--watch`) merges the K smallest blocks in a `(signal, date, level)` partition into one block at the next level via a DataFusion sort-merge, rebuilding sidecars (postings union, logs body bloom, metrics `series_types`), then supersede→(grace)→delete; the smoke script's logs/both legs assert the merge is lossless and inputs are reaped. Retention: the standalone `scry retention` tool reaps blocks whose newest record (`ts_max`) is past a per-signal TTL — **opt-in** (a signal with no configured TTL is never touched), **dry-run by default** (`--apply` required to delete); the smoke leg asserts dry-run is inert and `--apply` reaps logs while leaving untouched-signal blocks (metrics) in place. Metrics/logs preselect via postings; logs additionally skip whole blocks via the body bloom on `--grep`; traces/profiles push matcher/time/trace-id filters as parquet row predicates and prune on row-group stats. See `docs/decisions.md § D-029…D-039` for the rationale and the README milestone table for scope. The full-text bloom is one-sided (false positives cost a scan, never a missed match; exact `contains` is the backstop), case-sensitive, byte-trigram (n=3), and sized optimally per block at seal; see D-035. The engines remain Valkey-agnostic (they take a `&dyn Fence` + `&dyn BlockEventSink` only): compaction's `superseded_by IS NULL` query filter and `level` promoted into the `meta.json` sidecar (D-036); retention injects `now` and uses `mark_deleted` (+ the `deleted_at IS NULL` filter) for a correct grace window (D-037). The shared `delete_block_objects` helper lives in `scry-block`. **v0.9 makes them multi-instance** (D-038/D-039): two new crates — `scry-valkey` (the `ValkeyClient`, the `SET NX PX` + Lua compare-and-set lease replacing the unbuildable-on-Garage D-013 object-store lease, pub/sub, and a `ValkeySink`) and `scry-cluster` (the Valkey-agnostic convergence consumer + cursor poller + the generic-over-`LeaseProvider` maintenance loop). `scry ingest --mode full` runs the lease-guarded compaction + retention loops plus three-tier convergence (pub/sub → cursor poll → full-walk); `scry query` converges query-only; a peer-deleted block is healed at query time by `EvictOnNotFound` + one re-plan. Single-winner is a *correctness* requirement (blocks are UUID- not content-addressed), upheld by the commit-point fence (`meta.json` PUT last) + grace=0 immediate-delete; sealed by `MULTI=1 scripts/smoke.sh`. Still not implemented: partial-block rewriting at the TTL boundary (retention only drops wholly-expired blocks), orphan-object GC for uncommitted merge leftovers, catalog snapshot bootstrap, the multi-bucket pool, profiles flamegraph aggregation (pprof parse + stack-merge — deferred, Grafana renders pre-aggregated data), PromQL (demoted — own UI removes the Grafana-compat driver), and regex/case-insensitive full-text.

## Commands

Workspace build / test:

```bash
cargo build --release --workspace
cargo test --workspace
cargo test -p scry-proto fingerprint::tests::order_independence   # single test
```

Smoke-test the wire protocol end-to-end (two terminals):

```bash
./target/release/scry ingest --listen 127.0.0.1:4000
./target/release/noise-spewer --addr 127.0.0.1:4000 --rate 50 --duration 3s
```

The sink prints per-session counters on disconnect (`batches=… samples=… log_entries=… spans=… profiles=… rejected=0`).

Smoke-test the storage + query path end-to-end (single terminal — wraps the sink, spewer, `scry list` reconcile, a row-count assertion, and a `scry get` round-trip assertion):

```bash
scripts/dev-garage-up.sh         # one-time per machine
scripts/smoke.sh                  # default SIGNAL=dummy (v0.1 path)
SIGNAL=metrics scripts/smoke.sh   # v0.2: also asserts ≥1 block w/ postings + query round-trip
SIGNAL=logs    scripts/smoke.sh   # v0.4: same, for logs
SIGNAL=both    scripts/smoke.sh   # v0.4 exit criterion: metrics + logs through one sink
MULTI=1        scripts/smoke.sh   # v0.9: two scry ingest instances on one bucket + Valkey
```

The `MULTI=1` leg (`scripts/smoke-multi.sh`, dispatched from `smoke.sh`) needs a Valkey reachable at `SCRY_VALKEY_URL` (default `redis://127.0.0.1:6379`; a dev one comes up via `scripts/dev-valkey-up.sh`). It starts two `--mode full` daemons sharing one bucket, spews logs to both, and asserts (1) both catalogs converge to the same total live row count via pub/sub + cursor poll (no double-count), (2) exactly one instance wins each compaction partition (`level ≥ 1`, rows unchanged), and (3) coordinated retention reaps both catalogs to zero — with no panic / no "pass failed" in either daemon log.

**Dev Valkey (this machine).** A native host redis already binds `127.0.0.1:6379`, which collides with the host-networked `scripts/dev-valkey-up.sh` container. So there's a long-lived throwaway Valkey for scry dev work running as the docker container **`scry-valkey-smoke`** (`valkey/valkey:8.0-alpine`) on **`127.0.0.1:6380`**. Point scry at it with `SCRY_VALKEY_URL=redis://127.0.0.1:6380` (or `--valkey-url`). It's kept up between sessions for ongoing multi-instance work; if it's gone, recreate with `docker run -d --name scry-valkey-smoke -p 6380:6379 valkey/valkey:8.0-alpine`.

For `metrics`/`logs`/`both` the script additionally reconciles a fresh catalog and runs `scry get --signal <sig>` against it, asserting the queried row count equals the sink-accepted count — i.e. ingest → store → query is loss-free. Tweak via env vars: `BATCHES=20000 RATE=4000 scripts/smoke.sh` for stress runs. The script empties the dev Garage bucket on every run; don't point it at a bucket whose contents you want to keep.

The smoke script also wraps `scry ingest` in `/usr/bin/time -v` and prints a service-performance block at the end — peak RSS, user/sys CPU, records/sec, and **CPU-µs / record** (the headline regression sentinel; rate-independent, unlike `%CPU` which slides with the inter-batch idle gap). Full `time -v` output is kept in `$SMOKE_DIR/sink.time` for context (ctx switches, page faults, etc.). The script sends SIGINT directly to the `scry ingest` PID, not to `time`, because GNU time does not forward signals to its child.

Smoke-test the browser web UI end-to-end (builds the bundle + release `scry`, stub upstream, no Garage/Valkey):

```bash
scripts/smoke-webui.sh             # asserts SPA serve + auth + /api/query relay + logout
```

Run it for real against a live daemon:

```bash
(cd desktop && bun run build)
cargo build --release -p scry
SCRY_WEBUI_PASSWORD=secret ./target/release/scry web \
  --listen 0.0.0.0:8080 --queryd 127.0.0.1:4100              # then open http://<host>:8080/
```

Regenerate Rust bindings from the wire schema (only path that should touch `crates/proto/src/generated.rs` or `crates/binschema-runtime/src/*.rs`):

```bash
scripts/gen-proto.sh
# or, if binschema lives somewhere else:
BINSCHEMA_DIR=/path/to/binschema scripts/gen-proto.sh
```

The script shells out to a local checkout of binschema (`$HOME/Projects/binschema` by default). Both the generated bindings and the vendored runtime are committed so a normal `cargo build` never needs node or binschema installed.

## Workspace layout

Crates in `crates/`:

**Protocol**
- **`binschema-runtime`** — vendored binschema Rust runtime (bitstream, encoder/decoder context). Regenerated by `scripts/gen-proto.sh`; do not hand-edit.
- **`scry-proto`** — the wire protocol crate. Re-exports the generated types from `generated.rs` and adds four hand-written modules: `framing` (length-prefixed framing over async streams, 32 MiB cap), `constants` (Signal enum incl. `Dummy = 0xFE` for v0.1, ACK/REJECT/ERR/GOODBYE numeric codes, batch-size defaults), `fingerprint` (xxh3-64 over canonically-sorted labels, via `twox-hash`), and `build` (ergonomic `Frame`-returning constructors). The union carries a live-tail pair `Subscribe`(0x50)/`TailRecord`(0x51) and a merged-query live pair `LiveQuery`(0x52)/`LiveBatch`(0x53) alongside the ingest frames (D-052, D-054).
- **`scry-match`** — the shared Prometheus-style label-matcher crate (`LabelFilter`/`Matcher`/`MatchOp`): `key=value` | `key!=value` | `key=~regex` | `key!~regex` (whole-string-anchored regex, AND semantics). Lifted verbatim out of the agent's old `filter.rs` (D-043) so both the agent's `--keep` allow-list and the live-tail subscription filter (D-052) share one grammar. Depends only on `regex`/`anyhow`/`scry-proto::LabelPair` — kept out of `scry-proto` so `regex` doesn't leak into every proto dependent.

**Storage (v0.1)**
- **`scry-objstore`** — thin wrapper over apache `object_store` (`AmazonS3Builder`). `ObjStoreConfig::from_env()` reads `SCRY_OBJSTORE_*`. Path-style for Garage; HTTP allowed for localhost. No `put_if_absent` — Garage 1.0.x silently ignores `If-None-Match: *`, and v0.1's UUID v7 paths make conditional PUT unnecessary.
- **`scry-wal`** — per-writer, per-signal append-only WAL. `<dir>/<signal>/wal-<u64-seq>.log`, framed `[len:u32 BE][crc32:u32 BE][payload]`, 256 MiB segments, fsync on rotation, replay tolerates torn tails. `Wal::{open, append, rotate, mark_uploaded, replay}`. **Not internally synchronized** — wrap in `Arc<tokio::sync::Mutex<_>>` when sharing.
- **`scry-block`** — block builder + reader. `DummyBlockBuilder` buffers records column-shaped (three Vecs, sorted on close), serialises a single-row-group parquet (zstd-3), uploads parquet + JSON sidecar via `object_store::ObjectStore::put`. `BlockMeta` is the sidecar struct; `block_path` builds the canonical key (`<signal>/<yyyy>/<mm>/<dd>/<writer_id>/<block_uuid>.{parquet,meta.json}`).
- **`scry-catalog`** — rusqlite-bundled catalog. Schema verbatim from `ARCHITECTURE.md § The catalog § Schema` minus the `buckets` table (single-bucket v0.1). `Catalog::{open, insert_block, list_blocks, get_block, block_count, reconcile_from_bucket}`. WAL journal mode, partial indices on `(signal,date,ts_min,ts_max)` and `(bucket,signal,date,level)` where `deleted_at IS NULL`. **Snapshot bootstrap (D-055):** `snapshot.rs` — `save_snapshot` (`VACUUM INTO` a consistent copy on a *separate* read-only connection → PUT the single object `_catalog/snapshot.sqlite`, non-destructive/lease-free) + `restore_snapshot` (GET → `PRAGMA user_version` guard against `CATALOG_SCHEMA_VERSION` → atomic rename into place; returns `Restored{blocks} | NoSnapshot | VersionMismatch`). The schema version rides *inside* the db as `PRAGMA user_version` (stamped every `open`) because `init_schema` has no `ALTER` migrations — a version mismatch ⇒ the consumer discards the snapshot and full-reconciles. The `_catalog/` prefix is **reserved**: explicitly skipped by `reconcile_from_bucket` and the cluster `poll_once`/`full_walk` so a snapshot is never mis-parsed as a block sidecar.

**Server**
- **`scry-server`** — the ingest server as a library. Owns the TCP listener, handshake, Batch/Ping/Goodbye dispatch, the per-session counters, and the process-scoped `DummyPipeline` (WAL + active block builder + optional catalog, shared across sessions via `Arc<Mutex<_>>`, per `ARCHITECTURE.md § The WAL` — WAL is per-writer, not per-session). Pipeline is constructed by the caller and passed to `Server::new(...)`, so the single `scry` binary can share it with background uploaders or catalog-lookup code. `Server::serve_with_shutdown(shutdown_fut)` binds, accepts until the future completes, then flushes the pipeline once and returns. Also owns the live-tail `SubscriptionRegistry` (`tail.rs`, D-052) it threads into every connection handler — free when idle, taps the logs decode path only when a subscriber is registered. **Merged history+live query (D-054):** an optional retained recent-window `LiveRing` (`live_ring.rs`, logs-only, bounded by age + byte cap, fed off the logs phase-2 seam via a `RetainingLogsAppender` that stamps `(wal_shard, wal_seg)`) attached with `Server::with_live_ring`; the server answers `LiveQuery` frames on the ingest port by snapshotting the ring under the query predicate and replying one `LiveBatch`. The query-side merge lives in `QueryService` (`query_service.rs`): it fans `LiveQuery` out to the ingesters (discovered via an injected `&dyn LiveDiscovery` — `live_merge.rs`, keeping the crate Valkey-agnostic), dedups survivors against the catalog watermark with the pure `live_record_is_durable(seg, Option<H>)` selector (`None` ⇒ nothing durable ⇒ keep all — segments are 0-based, so absent ≠ `Some(0)`), and unions them with the block table as a `logs_live` `MemTable` behind a `CREATE VIEW logs`.

**Compaction & retention (v0.8)**
- **`scry-block::delete_block_objects`** — shared helper (lib fn in `crates/block/src/lib.rs`, next to `block_path`): deletes a block's parquet + meta.json + flagged sidecars, NotFound-tolerant. Used by both compaction (reaping merged-away inputs) and retention (reaping expired blocks).
- **`scry-retention`** — per-signal TTL retention engine (lib) + thin CLI (bin). `policy.rs` (`RetentionConfig` with opt-in `ttl_for`, `plan_reaping` selecting blocks whose `ts_max_unix_nano < now - ttl` — `now` injected for determinism); `engine.rs::retain_once` (dry-run reports candidates and touches nothing; `--apply` does optional `mark_deleted`+grace, then `delete_block_objects` → `delete_blocks`). `main.rs`: `--ttl`/`--ttl-<signal>` (opt-in, no implicit deletion), `--apply` (default off = dry-run), `--grace`, `--watch`. `parse_duration` adds `d` (days). See D-037.
- **`scry-compact`** — size-tiered compaction engine (lib) + thin CLI (bin). `policy.rs` groups live blocks (`catalog.list_blocks()`) by `(signal, date, level)` and plans one merge per partition with `>= fanout` blocks (the `fanout` smallest, into `level + 1`). `merge.rs::merge_blocks` reads the K input main parquets through a DataFusion `ORDER BY` (per-signal sort key), streams the sorted batches into a new parquet, and rebuilds sidecars (postings union via `scry_block::postings`, logs `body.bloom` via the streaming `BodyBloomBuilder`, metrics `series_types` union from input metas); uploads `main → [postings] → [bloom] → meta.json` (meta last). `engine.rs::compact_once` runs the per-merge lifecycle: merge → `insert_block` → `mark_superseded` → (grace) → `delete_block_objects` → `delete_blocks`. **Single-instance** (no compaction lease — deferred); grace defaults to 0 because `list_blocks` filters `superseded_by IS NULL`. The merge `SessionContext` sets `parquet.schema_force_view_types = false` so string columns stay `Utf8` (not `Utf8View`) — keeping the merged block schema-identical to a freshly-written L0 block. See D-036.

**Multi-instance (v0.9)**
- **`scry-block` seams** — three sync, dependency-free seams the engines depend on instead of Valkey: the `Fence` trait + `AlwaysValid` (a destructive step calls `fence.check()` and aborts if the lease was lost); the owned serde `BlockEvent { Created{meta} | Superseded{inputs,by,by_meta} | Deleted{signal,uuids} }` + `Envelope { origin, seq, event }` (the pub/sub wire shape — `Created` is byte-identical to a `meta.json`); and `BlockEventSink` + `NoopSink` (the upload/compaction/retention paths `emit(...)` into this).
- **`scry-valkey`** — the only crate that talks to Valkey (via `fred`). `ValkeyClient` (Arc handle, auto-reconnect); the **lease** (`SET key holder NX PX ttl` acquire, Lua compare-and-`PEXPIRE` auto-renew every `ttl/3`, latches `valid=false` before server-side expiry, Lua compare-and-`DEL` release) behind `LeaseProvider` + `ValkeyLeaseProvider`; **pub/sub** (`subscribe_blocks`, `parse_envelope`); `ValkeySink: BlockEventSink` (`spawn` → background publisher task); and the **tail ingester registry** (`registry.rs`, D-053): `TailRegistration::spawn` `SET`s `scry/tail/ingesters/<writer_uuid> = <advertised addr>` with a `PX` TTL renewed every `ttl/3` (server-side expiry, `Drop`/`deregister` compare-and-`DEL`), and `discover_tail_endpoints` reads them back via one read-only SCAN-in-Lua `eval`. Supersedes D-013's object-store lease, which Garage can't implement (no consensus).
- **`scry-cluster`** — Valkey-agnostic orchestration (unit-testable in-process with stubs): `apply_event` (idempotent catalog mutations — `INSERT OR IGNORE` / `mark_superseded` / `delete_blocks`), `poll_once` + `full_walk` (the cursor-poll and full-reconcile tiers, keyed on the `poll_cursors` high-water table), and the generic `run_maintenance_loop<L: LeaseProvider>` driving `run_compaction_pass` / `run_retention_pass` under a lease. `LocalLeaseProvider` (always-valid) backs `--allow-unfenced-maintenance` and the standalone CLIs.
- **`scry-catalog` cursors** — `poll_cursors(signal, writer_id, date, highest_uuid)` table + `get_cursor`/`advance_cursor` (monotonic-max UPSERT)/`list_cursors`, the high-water mark for incremental polling.
- **`scry-query::EvictOnNotFound`** — object-store decorator: a `NotFound` mid-scan parses the block UUID from the path, `delete_blocks` it from the stale catalog, and records the eviction; `scry-query`/`scry-queryd` do **one** transparent re-plan when a query failed with an eviction recorded.

**Binaries**

The ten operator roles are now a **single multicall binary `scry`** (crate `crates/scry`, D-049): a clap `Subcommand` enum dispatches `scry <role> …` to the corresponding role crate's `run(args)`, with one shared `tracing` init and one process-global mimalloc allocator. Each role below is now a **library crate** (`[lib]`, no `[[bin]]`) wrapped by that dispatcher; the crate names are unchanged. The two dev harnesses (`noise-spewer`, `scry-gateway-probe`) keep their own `[[bin]]`.

- **`scry`** — the dispatch binary (`crates/scry`): the clap `Subcommand` enum mapping `ingest`/`query`/`get`/`list`/`agent`/`gateway`/`web`/`compact`/`retention`/`tail` to the role crates' `run` entrypoints, the single `tracing` subscriber init, and the single mimalloc `#[global_allocator]`.
- **`noise-spewer`** — TCP client. Does the Hello handshake, then emits random metrics/logs/traces/profiles/dummy batches at a target rate, respecting `max_inflight_batches` from the HelloAck. Payload generators live in `gen.rs`. `--max-batches N` for an exact count (rate × duration is off-by-one).
- **`scry-ingestd`** (subcommand **`scry ingest`**) — the ingest server role; a thin layer around `scry-server`. Parses flags, optionally constructs a `DummyPipeline` from `SCRY_OBJSTORE_*` env + `--wal-dir [--catalog]`, then hands it to `Server::serve_with_shutdown(ctrl_c)`. All wire-protocol behaviour lives in `scry-server`. **v0.9:** when `--valkey-url`/`SCRY_VALKEY_URL` is set it builds a `ValkeyClient`, attaches a `ValkeySink` to the pipeline, and spawns the convergence consumer + cursor poller + full-walk; `--mode full` (vs `ingest-only`/`query-only`) additionally runs the lease-guarded compaction + retention maintenance loop (`--compact-interval`/`--compact-fanout`/`--compact-grace`, `--retention-interval`/`--ttl[-<signal>]`/`--retention-apply`/`--retention-grace`, `--lease-ttl`, `--no-maintenance`, `--allow-unfenced-maintenance`). Grace defaults to 600s with a lease present, 0 without. **v1 (D-054):** constructs the logs `LiveRing` (`--live-window-secs`, default 90; `--live-window-max-bytes`, default 128 MiB; `0` disables) and attaches it via `Server::with_live_ring`, so this ingester answers `LiveQuery` frames on its `--listen` port (discovered via the same `--tail-advertise-addr` Valkey registration as tail). **Catalog snapshot producer (D-055):** with an online catalog, `--catalog-snapshot-interval` (default 300s, `0` disables) spawns an independent bg task that periodically `save_snapshot`s the catalog (`VACUUM INTO` → PUT `_catalog/snapshot.sqlite`) so a cold `scry query` restores it in one GET instead of walking every sidecar. Non-destructive/lease-free — runs even with the maintenance loop paused.
- **`scry-queryd`** (subcommand **`scry query`**) — the query daemon role. **v0.9:** with `--valkey-url`/`SCRY_VALKEY_URL` it runs the same three-tier convergence (consumer + poller + full-walk) **query-only** — no maintenance loop, no lease contention. **v1 tail front-door (D-053):** an optional `--tail-listen <addr>` (separate from `--listen` — the tail `Frame` and query `QueryFrame` unions collide on their first byte) turns queryd into a **transparent live-tail relay** (`tail_relay.rs`): it discovers the live ingesters via `scry_valkey::discover_tail_endpoints`, dials each as a tail client (`scry_tail::dial_subscribe`), and fans their `TailRecord`s back to the `scry tail --queryd` client, re-discovering every `--tail-rediscover-interval` (5s). **No Valkey ⇒ refuse** each subscription with `ERR_TAIL_UNAVAILABLE` (9) rather than stream nothing. **Merged history+live query (D-054):** with Valkey it injects a `ValkeyLiveDiscovery` (over the same D-053 tail registry) into `QueryService.with_live_discovery`, so `scry query --live` (logs) unions the stored blocks with the ingesters' in-flight ring, deduped by the per-writer WAL-segment watermark; **no Valkey ⇒** a `--live` logs query is refused with `QUERY_ERR_LIVE_UNAVAILABLE` (0x0005) rather than a silent blocks-only degrade. **Catalog snapshot restore-on-boot (D-055):** on a cold boot (local catalog file absent), unless `--no-snapshot-restore`, it calls `restore_snapshot` — one GET of `_catalog/snapshot.sqlite`, `PRAGMA user_version`-guarded, atomic-renamed into place — instead of a full O(blocks) sidecar walk; `NoSnapshot`/`VersionMismatch`/any error ⇒ start cold and let the existing poll + full-walk loops converge.
- **`scry-query`** (subcommand **`scry get`**) — the one-shot query CLI role: opens a catalog and runs a single DataFusion query (`--signal`, `--trace-id`, `--grep`, …) directly against the bucket, printing rows. The local-query counterpart to the `scry query` daemon's wire.
- **`scry-list`** (subcommand **`scry list`**) — catalog inspector role. Opens a SQLite catalog (creates the schema if missing), runs `reconcile_from_bucket` unless `--no-reconcile`, prints one line per block + a `# total rows=N bytes=N` trailer.
- **`scry-agent`** (subcommand **`scry agent`**) — per-node k8s log collector: a filesystem scan of `/var/log/pods` spawns one tailer per CRI container log (`discovery.rs`/`cri.rs`), a Kubernetes pod watch enriches streams with `k8s_<podlabel>` labels (`stream.rs`), and a batcher ships `Signal::Logs` over the native wire (`scry-client`). **Node-side keep-only allow-list** (`filter.rs`, D-043): `--keep <matcher>` is repeatable and a stream is shipped only if it matches **ALL** matchers (logical AND); omitting `--keep` ships everything (opt-in, backwards-compatible). Each matcher is Prometheus-style — `key=value` | `key!=value` | `key=~regex` | `key!~regex` (regex whole-string-anchored, values may be double-quoted) — evaluated against the stream labels `namespace`/`pod`/`container`/`node` + `k8s_<key>`, with an absent label treated as empty (`key=~".+"` = "present"). The drop happens in `ingest()` right after labels are computed (before any batch state is touched, so dropped lines never go on the wire); the keep/drop verdict is cached per fingerprint so the (possibly regex) matchers run once per distinct stream, not per line. Dropped-line counter surfaces on a sparse `info!` + the final `agent done` log. **Prometheus metrics scraping** (`promparse.rs`/`scrape.rs`, D-045): the agent is also an Alloy-replacement for the metrics path — it pulls Prometheus `/metrics` endpoints and ships `MetricsBatch` over the *same* wire/connection (Hello declares `SIGNAL_BIT_LOGS | SIGNAL_BIT_METRICS` when scraping is active). Targets come from two sources unioned by `spawn_scrape_scheduler` (mirrors `spawn_log_scanner`: reconcile desired set → per-target scrape task → `metrics_tx` mpsc): **static** `--scrape-target <url>` (repeatable), and **discovered** — the pod watch (`apply_event`) builds a `ScrapeTarget` from any pod with `prometheus.io/scrape=="true"` (using `prometheus.io/{port,path,scheme}` + `status.podIP`), labeled `job`/`instance`/`namespace`/`pod`/`node` + `k8s_<label>` (the `stream_labels` convention; `job` = pod's `app.kubernetes.io/name`/`app`/`k8s-app` label, else pod name). The parser (`promparse.rs`) is **hand-rolled** (no prom-parser dep — Bart wants control over this layer): lenient text-exposition parse (HELP/TYPE, counter/gauge/histogram/summary/untyped, Go floats incl. NaN/±Inf, escaped label values, optional ms timestamps; malformed lines skipped + counted), with type resolution that strips `_bucket`/`_sum`/`_count` suffixes to the family. `scrape.rs::scrape_to_series` is the pure, unit-tested mapping → wire series: each series carries `__name__=<metric>` + the target labels (authoritative — a colliding exposed label is renamed `exported_<key>`, the Prometheus `honor_labels:false` default), fingerprinted with the same `fingerprint()` as the remote-write inbound. Every scrape synthesizes `up` (1/0) + `scrape_duration_seconds` gauges so a down target is data, not absence. The `--keep` allow-list applies to metric series too (matched against the full label set, cached per fingerprint). Auth is plain HTTP + optional bearer (`--scrape-bearer`, prefer `@/path` so the token isn't on argv). Flags: `--scrape-target`, `--scrape-bearer`, `--scrape-default-job` (static-target `job`), `--scrape-interval` (15s), `--scrape-timeout` (10s); scraping is enabled when there are static targets or the pod watch is on (so annotations are honored). Sealed by `scripts/smoke-agent-metrics.sh` (stub `/metrics` → agent → `scry ingest` → bucket → `scry get` asserts the 3 exposed + 2 synthesized rows land and query back). Known gaps vs Alloy (deferred): Service/Endpoints SD, per-target TLS/mTLS, native (sparse) histograms, exemplars/OpenMetrics extras, scrape WAL durability (at-most-once, like logs).

**TOML processing-pipeline config** (`config.rs`, D-047): the agent also accepts a `--config <path>` (`SCRY_AGENT_CONFIG` env) TOML file (usually ConfigMap-mounted) that owns six processing features. **Config owns the pipeline; flags own runtime** — connection, intervals, batch sizes, `--scrape-target` stay flags; per-signal `keep`, label manipulation, and JSON field extraction go in the file. The file structure is `[logs]`, `[logs.json]`, `[metrics]` with `deny_unknown_fields`. Features: (1) per-signal `keep` allow-list (supersedes the global `--keep`, which is rejected when `--config` is set); (2) `label_map` — surface a pod label (`k8s_<k>` → chosen name, suppressing the `k8s_` twin); (3) `static_labels` — inject fixed labels on every stream/series; (4) `json.labels` — promote JSON body fields to stream labels (fingerprinted → postings, low-cardinality); (5) `json.metadata` — promote JSON fields to per-entry attributes (the `Map<Utf8,Utf8>` column); (6) metric `label_map` — rename exposed metric keys before collision/`exported_` logic. Label precedence: core keys (`namespace`/`pod`/`container`/`node`) always win; static_labels beat json.labels beat label_map beat raw k8s_ keys. Sealed by `scripts/smoke-agent-config.sh` (CRI log fixture + stub `/metrics` + TOML config → agent → `scry ingest` → bucket → `scry get`, 10 assertions covering features 1/3/4/5/6; feature 2 proven by unit test).

**Kubelet/cadvisor scraping + label-selector pod SD** (`config.rs`/`scrape.rs`/`discovery.rs`, D-048): two more metrics-SD pieces for k8s parity, both config-owned (`[metrics]` block) per D-047. **Kubelet scraping** (`[metrics.kubelet]`): scrape *this* node's kubelet over HTTPS (`https://${NODE_IP}:10250` by default; `${NODE_IP}`/`${NODE_NAME}` interpolated at startup from the new `--node-ip`/`NODE_IP` downward-API flag), emitting one target per enabled endpoint — `/metrics/cadvisor` (`job=cadvisor`, per-container metrics) + `/metrics` (`job=kubelet`, kubelet-self), both on by default, individually togglable. TLS is configurable, **default `insecure_skip_verify=true`** (the kubelet cert rarely has a SAN for the dialed IP — matches Prometheus' stock job; `ca_file` opts into verification). The bearer comes from a `bearer_file` (default the SA-token path) **re-read per scrape** so projected-token rotation is followed. **Pod-label SD** (`[[metrics.scrape_pods]]`, repeatable): scrape pods whose labels match a `selector` (`matchLabels` AND semantics) — node-local, reusing the existing node-field-selected pod watch with **no new pod RBAC**; annotation SD (`prometheus.io/scrape`) still takes precedence. Mechanically: `ScrapeTarget.bearer` is now a `BearerSource` (`Literal | File`, file re-read in `fetch`) + a per-target `TlsProfile` (`{insecure_skip_verify, ca_file}`); a `ClientPool` lazily builds one `reqwest::Client` per distinct `TlsProfile` (reqwest bakes TLS at build time — standard targets share one verify-on client, kubelet adds one skip-verify client). `scrape_to_series`/`relabel_scrape`/`stamp_target` stay **frozen**; new helpers `pod_matches`/`assemble_pod_target`/`build_kubelet_targets`/`resolve_kubelet_address`. Kubelet scraping needs a new RBAC rule (`nodes/metrics` + `nodes/proxy`, GET — kubelet delegates authz to a SubjectAccessReview). Manifests updated: `deploy/k8s/agent-rbac.yaml` (the rule), `deploy/k8s/agent-daemonset.yaml` (`NODE_IP` env from `status.hostIP` + `--config /etc/scry/agent.toml` + ConfigMap mount), new `deploy/k8s/agent-config.example.yaml`. Sealed by `scripts/smoke-agent-kubelet.sh` (self-signed HTTPS + bearer-gated stub serving both endpoints → `[metrics.kubelet]` config → agent → `scry ingest` → bucket → `scry get`, asserting 6 rows land, both endpoints scraped, both scrapes auth'd over TLS-skip-verify, and a static `cluster` label rides all rows; needs Garage + `openssl`/`aws`/`sqlite3`/`python3`). Pod-label SD is proven by `pod_matches`/`build_scrape_target` unit tests (the smoke omits the pod watch). Remaining gaps vs Prometheus/Alloy (deferred): Service/Endpoints SD, per-job TLS for pod-SD targets, mTLS client certs, hot config reload.

**Gateway (fan-out hub)**
- **`scry-gateway`** (subcommand **`scry gateway`**) — a **fan-out hub**: it decodes records from several *inbound* protocols into a typed `*Batch` and tees each one, best-effort, to every configured *outbound* sink. **Inbounds** (all decode to the same typed batch): the foreign HTTP routes — `otlp.rs` (`POST /v1/traces`, OTLP/HTTP protobuf), `pyroscope.rs` (`POST /ingest`), `promwrite.rs` (`POST /api/v1/write` + `/api/v1/push`, remote-write v1 snappy+protobuf) — plus, opt-in, `wire.rs`, a trimmed native binschema listener (Hello/HelloAck + Batch/Ping/Goodbye, a deliberate copy of `scry-server`'s handshake minus the storage pipeline; `scry-server`'s `Server` is hardwired to `ShardedPipeline` and can't be reused as a pluggable sink). **Fan-out core** (`sink.rs`): `AppState` holds `Arc<Vec<SinkHandle>>`; `offer_{logs,metrics,traces,profiles}` Arc-wrap the batch once and offer it to every `SinkHandle` whose `accepts` signal-mask matches. Each `SinkHandle` is a bounded `mpsc` queue + a worker task (`spawn_sink`); `offer` is `try_send` — on a full/closed queue it drops + counts (logged on a sparse cadence), never blocking the inbound or the other sinks. **Outbound sinks** (each opt-in; at least one required): `sink_scry.rs` (`ScrySink`, accepts all four signals, **connects lazily inside its worker** from a `ScryConnect` — a down/absent upstream at startup is not fatal — zstd-3 + `build::batch` + one reconnect-and-resend on failure, forgets a dead client so the next item reconnects); `loki.rs` (`LokiSink`, logs only — pure `to_push_request` maps one Loki stream per `LogStream`, label keys sanitized to `[a-zA-Z_][a-zA-Z0-9_]*`, severity + entry attributes as structured metadata, POST `{url}/loki/api/v1/push`); `opensearch.rs` (`OpenSearchSink`, logs only — **self-managing, per-service**: `--opensearch-index` is a *prefix*, and each `LogStream` is bulk-`create`d into the **per-service rolling data stream** `<prefix>-<service>` where `<service>` is the first present of `service.name`/`service`/`app`/`k8s_app` (sanitized) or `general`; doc per entry with ISO-8601 millis `@timestamp` + `flat_object` labels/attributes, POST `{url}/_bulk`). Unless `--opensearch-unmanaged`, the sink **owns the lifecycle assets** so cluster-side drift can't silently break ingest (D-042): it asserts — at startup, on `--opensearch-reconcile-interval`, **and** after a write error (not boot-only) — an **ISM rollover policy** `<prefix>-rollover` (roll by `--opensearch-rollover-size`/`-age`, **no auto-delete**, auto-attached via `ism_template.index_patterns ["<prefix>-*"]` so rolled-over backing indices stay managed), an **index template** `<prefix>` (`data_stream` enabled, `flat_object` label mappings — the fix for the mapping-explosion/type-conflict drops that silently drop docs), and the per-service data streams (lazily, tracked in a worker-local set). The ISM policy is drift-corrected read-then-write (GET `_seq_no`/`_primary_term` → conditional PUT); the template PUT is idempotent. With `--opensearch-aws-sigv4` (for Amazon OpenSearch Service / Serverless, which reject unsigned requests) the sink signs **every** request — bulk writes *and* management calls — with AWS SigV4 via `aws_sign.rs` (`SigV4Signer`): credentials + region come from the standard chain (`aws-config`: env / shared profile / EKS IRSA / EC2-ECS IMDS), the signing name is `--opensearch-aws-service` (`es` for managed domains, `aoss` for Serverless), and creds are cached in the single-threaded worker until near expiry; `x-amz-content-sha256` is emitted so the body is covered. `mimir.rs` (`MimirSink`, **metrics only** — the inverse of the remote-write inbound: pure `to_write_request` reverses `map_remote_write`, building a `fingerprint→labels` map from the batch's series dict, grouping `samples` by fingerprint into one `TimeSeries` each, ns→ms timestamps, dropping samples with no dict entry; reuses `promwrite`'s prost types + `encode_snappy` so encode is symmetric with decode; POST `{url}/api/v1/push` with `Content-Type: application/x-protobuf`, `Content-Encoding: snappy`, `X-Prometheus-Remote-Write-Version: 0.1.0`, and `X-Scope-OrgID` when `--mimir-tenant` set; D-044). All HTTP sinks (Loki/OpenSearch/Mimir) share one `reqwest::Client` built by `tls.rs::build_http_client`, which optionally **adds** a custom CA (`--ca-cert`, PEM bundle via `Certificate::from_pem_bundle`) on top of the built-in roots (the scry sink is native TCP, no TLS). **All in → all out, zero routing config** — for anything selective, run a second gateway. `main.rs` flags: `--listen` (HTTP, default `0.0.0.0:4318`), `--listen-wire` (native, **opt-in** `Option` — no default bind), `--upstream` (scry, **opt-in** `Option` — no default; the scry sink exists only when set), `--loki-url`, `--opensearch-url`/`--opensearch-index` (prefix)/`--opensearch-unmanaged`/`--opensearch-rollover-size` (`30gb`)/`--opensearch-rollover-age` (`1d`)/`--opensearch-reconcile-interval` (`5m`)/`--opensearch-aws-sigv4`/`--opensearch-aws-region`/`--opensearch-aws-service` (`es`), `--mimir-url`/`--mimir-tenant` (metrics out), `--ca-cert` (custom CA for the HTTP sinks), `--sink-queue-cap`, `--sink-http-timeout`; **at least one sink must be configured** (else a startup `bail!`); graceful shutdown on SIGINT **or** SIGTERM (k8s). The pure mappers/builders (`map_traces`, `map_remote_write`, `to_push_request`, `to_bulk_ndjson`, `data_stream_name`, `ism_policy_doc`, `index_template_doc`) are unit-tested; the smoke is `scripts/smoke-gateway.sh` (HTTP inbounds + scry sink). Consequence (D-041): the inbound ACKs on **enqueue**, not on downstream confirm — durability across an outage is bounded by each sink's queue depth.

**Web UI**
- **`scry-webui`** (subcommand **`scry web`**) — standalone axum 0.8 server that serves the SolidJS query app to a browser and relays queries to a `scry query` daemon, so the UI is reachable remotely without the desktop binary. It is a **dumb byte-pipe**, exactly like the Tauri shell: `POST /api/query` dials one of the server's *own* configured upstreams, writes the already-framed request, reads to EOF, returns the bytes. Zero protocol knowledge — the whole wire protocol lives in the TypeScript frontend. **Multi-target (D-046):** `--queryd` is repeatable as `id=host:port` (first = default; a single bare `host:port` keeps id `default`); the server holds a `Vec<Target{id,label,addr}>` allowlist where `addr` is `#[serde(skip)]` (never reaches the browser). `GET /api/targets` (auth-gated) lists `{id,label}` + the default; `POST /api/query` picks the upstream from the **`X-Scry-Target`** header (a target *id* resolved server-side — absent/empty ⇒ default, unknown ⇒ **400**, unreachable ⇒ 502). The browser only ever sends an opaque id, never a raw address — **SSRF-safe**. The frontend always shows a target `<select>` in browser mode. Auth is a single shared password (`SCRY_WEBUI_PASSWORD`, never argv) → a signed session cookie (`axum-extra` `SignedCookieJar`, key HKDF-derived from the password so sessions survive a restart). The SolidJS bundle is embedded via `rust-embed` (single self-contained binary in release; reads `desktop/dist` from disk in debug — run `bun run build` in `desktop/` before `cargo build -p scry`). `src/{main,lib,auth,query,assets}.rs`; integration tests in `tests/{auth,query}.rs`. **Home-machine deployment only — deliberately not in the Docker image / k8s manifests.** See D-040, D-046.
- **`scry-tail`** (subcommand **`scry tail`**, crate `crates/tail`) — the live log-tail client (D-052/D-053). Two endpoint flags, wire-identical from the client's side (both speak the tail sub-protocol): **`--queryd <host:port>`** (the **primary/blessed path** — the query daemon's `--tail-listen`, which discovers ingesters via Valkey and relays) and **`--ingest <host:port>`** (direct-to-ingester, for Valkey-free debugging); both repeatable, ≥1 required, trailing positional args are `scry-match` matchers. Does the Hello handshake, sends one `Subscribe`, and prints every `TailRecord` the endpoint sprays back — `<rfc3339 ts> <LEVEL> {labels} body` to **stdout** (the data stream), all diagnostics to **stderr** via `eprintln!` (deliberately *not* `tracing`, which the shared `scry` subscriber routes to stdout). The connect→Hello→Subscribe→read-loop core is the reusable `pub async fn dial_subscribe`, shared with the queryd relay (D-053). Best-effort/live-only: never merged with stored history, never deduped, lossy under load. **v1: logs only.** Server side lives in `crates/server/src/tail.rs`: a process-local `SubscriptionRegistry` (an `AtomicUsize` subscriber count the logs ingest path reads once per batch — zero cost when nobody's tailing) + a `TappingLogsAppender` `LogsAppender` decorator that forwards matching entries via bounded-channel `try_send` (drop-on-full, never backpressures ingest) while delegating storage unchanged. An ingester with `--valkey-url` set advertises its tail address (resolved from `--tail-advertise-addr`/concrete `--listen`/`NODE_IP`, else a startup error on a wildcard bind) into the Valkey registry so the queryd front-door can find it. Sealed by `scripts/smoke-tail.sh` (direct path, self-contained) + `scripts/smoke-tail-queryd.sh` (front-door path, needs only a dev Valkey). See D-052, D-053.
- **`desktop/`** — the Tauri + SolidJS desktop query app (not a workspace member; heavy tauri/webkit deps). The frontend is **dual-mode**: the `Transport` interface (`src/protocol/transport.ts`) has two impls — `transport-tauri.ts` (native socket) and `transport-http.ts` (`fetch` to scry-webui's `/api/query`) — picked at runtime by `isTauri()` (`src/env.ts`) via a dynamic `import()`, so the browser bundle never imports `@tauri-apps`. ## Architecture you have to read multiple files to grasp

The wire protocol is a **flat tagged union, not header+payload.** Each `Frame` IS a discriminated union whose first byte is the tag matching one of nine `FrameMsg` variants (Hello/HelloAck/Batch/BatchAck/FlowControl/Ping/Pong/Goodbye/Error). Framing is `[len: u32 BE][Frame]` with a 32 MiB cap (`framing::MAX_FRAME_BYTES`). There is intentionally no separate `protocol.header` section in the schema — binschema's header+payload model didn't fit, and we dropped it. If you find yourself wanting one, read the schema's comments first.

The schema (`proto/ingest.schema.json`) is the source of truth for both formats and constants. `crates/proto/src/constants.rs` mirrors the numeric values by hand so call sites can `match` on them; if you change a code in the schema, mirror it there. The schema currently has **zero `computed` fields** (`length_of`, `crc32_of`, …). `crates/proto/src/build.rs` relies on this: it constructs `Frame`s via `XxxInput { ... }.into()`, which populates `const` fields (tag bytes) from the generated `From` impls. If a `computed` field is ever added, `.into()` will silently fill it with the codegen default — the on-the-wire bytes are still correct (the encode path recomputes them), but the in-memory `Output` struct will be misleading. The caveat is documented at the top of `build.rs`; audit every constructor there before merging such a change.

Series-dictionary compression: every signal that carries labels (`MetricsBatch.series`, `LogsBatch.streams`, etc.) references a per-batch dictionary entry by `fingerprint: u64` rather than re-shipping labels per sample. The fingerprint is xxh3-64 over `key\0value\0`-joined, lexicographically sorted `(key, value)` pairs (`fingerprint::fingerprint`). The same function must be used on the encode and decode side or fingerprints won't match.

Batches are zstd-compressed at level 3 by the spewer; the sink decompresses based on `Batch.compression` (`COMPRESSION_NONE = 0`, `COMPRESSION_ZSTD = 1`). `uncompressed_size` is on the wire so the sink can pre-size its buffer; `record_count` is on the wire so the sink can sanity-check before decoding.

The big-picture architecture for the eventual storage/query system — block layout, bucket pool, catalog, WAL, ingest/query split, per-signal specialisations, resource isolation — is in `docs/ARCHITECTURE.md` (~2000 lines). Architectural decisions with rationale (D-001 … D-039) are in `docs/decisions.md`. Read both before changing anything beyond the noise harnesses; in particular D-027 (resource isolation between signals and workloads) and D-028 (profiling as a development principle) shape any future hot-path code.

## Conventions

- **Big-endian everywhere on the wire.** `xxd` on a captured stream should be human-readable.
- **Reasons are numeric.** Free-text `message` fields in BatchAck/Goodbye/Error are operator-log only. Both sides decide based on the numeric code.
- **Versioning lives in the handshake.** `Hello.protocol_version` is the only negotiation point; no per-message version bytes. Bump it and let the server take `min(agent, server)`.
- **`unsafe_code = "forbid"`** workspace-wide (`Cargo.toml`).
- **Release profile uses thin LTO and `codegen-units = 1`.** Don't override casually — the harnesses are sized to give realistic numbers.
- **One crate version, kept in lockstep with the git tag.** Every first-party crate inherits `version` from `[workspace.package]` (`version.workspace = true`); the lone exception is the vendored `binschema-runtime`, which pins its own version to track upstream binschema. Cut a release with `scripts/release.sh <version>` — it bumps that single field, refreshes `Cargo.lock`, commits `chore(release): vX.Y.Z`, and creates the annotated `vX.Y.Z` tag (it does **not** push). This is what makes the `CARGO_PKG_VERSION` baked into block `producer_version` and the wire `agent_version` actually match the release tag, so don't hand-edit per-crate versions or tag without bumping.
